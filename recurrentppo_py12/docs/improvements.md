# recurrentppo_py12 の改善状況と残課題

日経225トレーディングの改善ロードマップ。qrdqn_py12 のレビュー（B/F/G 項番）を引き継ぎ、
本バージョンでの対応状況と、これからやるべきことを整理する。

---

## 1. 本バージョンで対応済み

| 項目 | 対応内容 | 実装場所 |
|---|---|---|
| **G-1** アルゴ変更 | QR-DQN → **RecurrentPPO**（LSTM内蔵, sb3-contrib）。オンポリシーで安定。`ALGO="ppo"` で通常PPOと比較可 | `algo.py` |
| **G-1** 観測の再設計 | 「130日×29特徴のwindow」→ **1日1ベクトル（29特徴＋ポジションone-hot 3）**。時系列の記憶は内蔵LSTMに委譲。取引コストにより最適行動が現在ポジションに依存するため、ポジションを観測に明示 | `main.py NikkeiEnv` |
| **B-3** 正規化の刷新 | ウィンドウ内MinMax（絶対水準破壊・基準ズレ）を廃止 → **リターン化/相対化＋ローリング・ロバスト z-score**（中央値/MAD, 窓252日, 因果的=未来を見ない, ±5クリップ） | `NikkeiEnv._precompute_features` |
| **G-2** 正則化 | **weight decay**（`WEIGHT_DECAY=1e-4`）を Adam に適用。アーキはLSTM一本に簡素化（MLP/ResNet等の横断比較は qrdqn_py12 側で実施済 → [[arch-comparison-mlp-vs-resnet]]） | `algo.py build_model` |
| **G-3** 複数OOS窓評価 | `_eval_one.py` が `EVAL_WINDOWS`（コロナ2020-22 / 軟調2022-24 / bull2024-）で**窓ごとに B&H と比較**し、超過リターン×シャープのマトリクスと「全窓edge」判定を出力 | `_eval_one.py` / `config.EVAL_WINDOWS` |
| **F-4** 当日データ投入 | `data.fetch_latest()`（yfinance/FRED で最新OHLC・VIX・金利）＋ `server.py` の「📥 最新データ自動取得」ボタン。手打ちは上書き用途に限定 | `data.py` / `server.py` |
| **D-2** 合成行のATR歪み | F-4 と同時解消。manual_data を実OHLC行で延長（High=Low=Open の合成行を廃止）。重複日付は手動データ優先 | `data.py generate_env_data` / `run_simulation.py` |
| 評価時のLSTM文脈 | OOS開始位置へジャンプする評価では、**開始前 `WINDOW_SIZE` 本の観測をLSTMに流して隠れ状態を温めてから取引開始**。全評価系（`_eval_one` / `_eval_ensemble` / `run_simulation` / `visualize`）がこれを使用 | `main.warmup_lstm_state` / `main.rollout` |
| 検証選抜のLSTM文脈 | 学習中の checkpoint 選抜も同条件に統一。標準 `EvalCallback`（ゼロ状態で検証窓を走る）を **`WarmupEvalCallback`**（`rollout` と同じ warmup 手順で1エピソード評価）に置換。「選抜時の条件 = 使用時の条件」になった | `main.WarmupEvalCallback` |

パイプラインは通し確認済み（短い学習 → 複数窓評価 → アンサンブル → 単日シミュレーション）。
**本学習（500kステップ・複数シード）はまだ回していない。**

## 2. 次のアクション（優先順）

### N-1. 本学習＋複数シードで G-3 の判定を出す（最優先）
- `python main.py <seed>` をシード5本以上（Colab GPU推奨, `notebooks/train_colab.ipynb`）。
- `_eval_one.py` の窓×モデルのマトリクスで「全窓edge」が出るかを見る。
- 判定基準は1窓の絶対リターンではなく、**窓×シード横断での超過リターン/シャープの分布**。
  - 全窓edge=◎ が安定して出る → エッジ有り。アンサンブルで運用検討。
  - 出ない → 「買い持ちが正解」の公算大。目的を「bullで勝つ」から「bear/暴落でDDを抑える」に切り替えて評価軸を再定義する。
- 注意: 2024-01 より前の窓は in-sample 参考値。純OOSは bull 窓のみなので、結論には qrdqn_py12 と同じ「強い上昇相場1窓くじ」問題が部分的に残る。**学習終端を 2020/2022 にずらした再学習（ローリングOOS）が本来の完成形**（→ N-4）。

### N-2. 検証選抜のレジーム不一致への対処
- qrdqn_py12 で観測した根本問題: val(2022-06〜2024-01, 軟調) と test(bull) の地合い不一致で、検証ベスト選抜が test で機能しない。
- 前提条件のズレ（選抜はゼロLSTM状態・本番はwarmup済み）は `WarmupEvalCallback` で解消済み（上表）。
- ただし val 窓が軟調1窓のままという**地合い不一致の本体は未解決のまま持ち越し**。
- 候補:
  - 検証窓を複数（軟調＋上昇の2窓）にして平均スコアで選抜する EvalCallback 拡張。
  - 検証指標を「対B&H超過のシャープ（情報レシオ）」にする（地合いに不変な指標で選ぶ）。
- ここを直さない限り「checkpoint選抜の運ゲー」は続く（[[try19-oos-findings]] のcheckpoint分散問題と同根）。

### N-3. アンサンブル運用の標準化
- 単体checkpointの分散はアンサンブル（多数決）で平均化する方針（`_eval_ensemble.py` 実装済み）。
- 残作業: シードN本×検証ベストのzipを `models/seed*/` に揃え、`_eval_one.py` と同様に**複数窓でアンサンブルを評価**する（現状アンサンブル評価は2024-窓のみ）。

### N-4. ローリングOOS（学習終端をずらした再学習）
- TRAIN_END を 2020-01 / 2022-01 / 2024-01 に変えて3セット学習し、各セットの「直後2年」を純OOSとして評価。
- これで初めて3窓すべてが純OOSになり、エッジ判定が統計として成立する。
- `config.TRAIN_END` がモデル名に入る設計なのでセット管理は容易。

### N-5. RecurrentPPO固有のチューニング（N-1 の結果を見てから）
- `LSTM_HIDDEN`（128→64: 過学習抑制方向）、`ENT_COEF`（0.01→0.005〜0.03: Flat張り付き対策）、`N_STEPS`/`BATCH_SIZE`。
- 学習エピソードが常に同じ日から始まる（trade_start固定）ため、LSTMが「経過日数」を暗記するリスクがある。気になる場合はエピソード開始位置のランダム化を検討（reset時に trade_start〜n の範囲でランダムスタート）。
- 報酬は現状 `dsr`。qrdqn_py12 では `excess_dsr` が最良（sharpe≈1.0）だったので、RecurrentPPO でも `REWARD_TYPE="excess_dsr"` の比較は早めに取る価値あり。

## 3. 中長期の候補（据え置き）

### SAC / TD3 による連続ポジションサイズ
- 行動を連続値(-1〜+1)にしてポジション強度を学ばせる（全力ロング/ショートの二択をやめる）→ 過剰トレード抑制・DD低減。
- env の action空間変更＋約定ロジック変更が必要（移行コスト高）。離散3行動でエッジが確認できてから。

### dropout の追加（G-2 残り）
- weight decay は導入済み。LSTM層間 dropout（`n_lstm_layers>=2` が前提）や policy head への dropout は未導入。
- 過学習兆候（train/val 乖離）が TensorBoard で見えたら入れる。

### 特徴量の見直し
- B-3 の正規化刷新で観測の質は上がったが、特徴セット自体（29本）は qrdqn 時代のまま。
- 冗長なσバンド6本×2系統などは LSTM には情報過多の可能性。相関の高い列の間引き比較は低コストで試せる。

## 4. 前提認識（G-0, 変わらず重要）

- 金融データは **S/N比が極端に低い**（日次リターンはほぼランダムウォーク）。「不安定・負ける」のは設計ミスというより問題の本質も大きい。
- **アルゴ/アーキの交換だけで黒字化はしない**。安定性は「①アルゴリズム ②報酬設計 ③正則化・検証 ④アンサンブル」の合わせ技で出る。本版は①②③に手を入れた。残りの伸びしろは**検証選抜（N-2）とアンサンブル（N-3）、そして評価の統計的土台（N-1/N-4）**にある。
